# Task 1.1: Dangerous Command Guard

Navigation: [Design](../design.md) | [PRD](../../../stories/1/prd.md)

## Objective

Create `.claude/hooks/dangerous-command-guard.sh` - a Bash script that intercepts Bash tool calls and either blocks dangerous commands or requires confirmation for risky ones.

## Output File

`.claude/hooks/dangerous-command-guard.sh`

## Input Specification

JSON on stdin:
```json
{
  "tool_name": "Bash",
  "tool_input": {
    "command": "the shell command to execute"
  }
}
```

## Output Specification

| Scenario | Exit Code | stdout | stderr |
|----------|-----------|--------|--------|
| Allow | 0 | (empty) | (empty) |
| Ask | 0 | JSON with `permissionDecision: "ask"` | (empty) |
| Block | 2 | (empty) | Block message with pattern + command |

## Implementation

### Constants

Define two arrays at the top of the script:

#### DENY_PATTERNS (Hard Block)
```bash
deny_patterns=(
    # Filesystem destruction
    'rm\s+-rf\s+/'
    'rm\s+-rf\s+\*'
    'rm\s+-rf\s+~'
    '>\s*/dev/sd'
    'mkfs'
    'dd\s+if=.*of=/dev/'

    # Git disasters
    'git\s+reset\s+--hard\s+origin'
    'git\s+push.*--force.*main'
    'git\s+push.*--force.*master'
    'git\s+push.*-f.*main'
    'git\s+push.*-f.*master'

    # Database destruction
    'DROP\s+DATABASE'
    'DROP\s+SCHEMA.*CASCADE'
    'TRUNCATE.*CASCADE'

    # Docker nuclear options
    'docker\s+system\s+prune\s+-a.*--volumes'
    'docker\s+system\s+prune.*--volumes.*-a'
    'docker\s+volume\s+prune\s+-f'
    'docker\s+volume\s+prune\s+--force'

    # Shell bypass of ALWAYS_BLOCK files (prevents using Bash to modify protected files)
    # .env files
    '(echo|cat|printf)\s+.*>\s*.*\.env'
    'tee\s+.*\.env'
    'sed\s+-i.*\.env'
    'ed\s+.*\.env'
    '(cp|mv)\s+.*\s+.*\.env'
    # Private keys
    '(echo|cat|printf)\s+.*>\s*.*\.(pem|key)'
    '(echo|cat|printf)\s+.*>\s*.*id_(rsa|ed25519)'
    'tee\s+.*(\.pem|\.key|id_rsa|id_ed25519)'
    'sed\s+-i.*(\.pem|\.key|id_rsa|id_ed25519)'
    '(cp|mv)\s+.*\s+.*(\.pem|\.key|id_rsa|id_ed25519)'
    # Secrets files
    '(echo|cat|printf)\s+.*>\s*.*(secrets\.(yml|yaml)|credentials\.json|service-account\.json)'
    'tee\s+.*(secrets\.(yml|yaml)|credentials\.json|service-account\.json)'
    'sed\s+-i.*(secrets\.(yml|yaml)|credentials\.json|service-account\.json)'
    '(cp|mv)\s+.*\s+.*(secrets\.(yml|yaml)|credentials\.json|service-account\.json)'
    # Git/SSH config
    '(echo|cat|printf)\s+.*>\s*.*\.git/config'
    '(echo|cat|printf)\s+.*>\s*.*\.ssh/'
    'sed\s+-i.*\.(git/config|ssh/)'
)
```

#### ASK_PATTERNS (Require Confirmation)
```bash
ask_patterns=(
    # File deletion
    'rm\s+-rf'
    'rm\s+-r'
    'rm\s+.*\*'

    # Git operations that push/modify remote
    'git\s+push'
    'git\s+reset\s+--hard'
    'git\s+clean\s+-fd'
    'git\s+checkout\s+--\s+\.'

    # Package publishing
    'npm\s+publish'
    'yarn\s+publish'
    'pip\s+upload'
    'cargo\s+publish'

    # Docker - volume/data destruction
    'docker-compose\s+down.*-v'
    'docker-compose\s+down.*--volumes'
    'docker\s+compose\s+down.*-v'
    'docker\s+compose\s+down.*--volumes'
    'docker\s+volume\s+rm'
    'docker\s+volume\s+prune'
    'docker\s+system\s+prune'
    'docker\s+rm.*-v'
    'docker\s+rm.*--volumes'
    'docker\s+container\s+prune'
    'docker\s+image\s+prune\s+-a'

    # Docker - stopping/removing services with data
    'docker\s+stop'
    'docker\s+kill'
    'docker-compose\s+down'
    'docker\s+compose\s+down'

    # Database operations
    'DROP\s+TABLE'
    'TRUNCATE'
    'DELETE\s+FROM.*WHERE\s+1=1'
    'DELETE\s+FROM[^W]*$'

    # Service restarts
    'systemctl\s+stop'
    'systemctl\s+restart'
    'service\s+.*\s+stop'

    # Kubernetes
    'kubectl\s+delete'
    'kubectl\s+drain'
    'helm\s+uninstall'

    # Shell bypass of REQUIRE_CONFIRMATION files (prevents using Bash to modify protected files)
    # Lock files
    '(echo|cat|printf)\s+.*>\s*.*(package-lock\.json|yarn\.lock|pnpm-lock\.yaml)'
    'tee\s+.*(package-lock\.json|yarn\.lock|pnpm-lock\.yaml)'
    'sed\s+-i.*(package-lock\.json|yarn\.lock|pnpm-lock\.yaml)'
    # Docker files
    '(echo|cat|printf)\s+.*>\s*.*(Dockerfile|docker-compose\.(yml|yaml))'
    'tee\s+.*(Dockerfile|docker-compose\.(yml|yaml))'
    'sed\s+-i.*(Dockerfile|docker-compose\.(yml|yaml))'
    # CI/CD files
    '(echo|cat|printf)\s+.*>\s*.*\.github/'
    '(echo|cat|printf)\s+.*>\s*.*\.gitlab-ci\.yml'
    'sed\s+-i.*\.(github/|gitlab-ci\.yml)'
    # Build config
    '(echo|cat|printf)\s+.*>\s*.*(Makefile|tsconfig\.json|pyproject\.toml|Cargo\.toml)'
    'tee\s+.*(Makefile|tsconfig\.json|pyproject\.toml|Cargo\.toml)'
    'sed\s+-i.*(Makefile|tsconfig\.json|pyproject\.toml|Cargo\.toml)'
    # Claude config
    '(echo|cat|printf)\s+.*>\s*.*\.claude/'
    'tee\s+.*\.claude/'
    'sed\s+-i.*\.claude/'
)
```

### Algorithm

```
1. Parse command from stdin JSON using jq
2. Loop through deny_patterns:
   - If command matches (grep -Eiq):
     - Write block message to stderr
     - exit 2
3. Loop through ask_patterns:
   - If command matches (grep -Eiq):
     - Write ask JSON to stdout
     - exit 0
4. exit 0 (allow)
```

## Dependencies

- `jq` - for JSON parsing
- `grep -E` - for extended regex matching

## Testing

```bash
# Test block
echo '{"tool_input":{"command":"rm -rf /"}}' | ./dangerous-command-guard.sh
echo "Exit code: $?"  # Should be 2

# Test ask
echo '{"tool_input":{"command":"git push origin main"}}' | ./dangerous-command-guard.sh
echo "Exit code: $?"  # Should be 0, with JSON output

# Test allow
echo '{"tool_input":{"command":"ls -la"}}' | ./dangerous-command-guard.sh
echo "Exit code: $?"  # Should be 0, no output
```

## Acceptance Criteria

- [ ] Script is executable (`chmod +x`)
- [ ] Shebang is `#!/usr/bin/env bash`
- [ ] Uses `set -euo pipefail` for safety
- [ ] All deny_patterns from design are implemented
- [ ] All ask_patterns from design are implemented
- [ ] Shell bypass patterns for ALWAYS_BLOCK files are implemented (echo/cat/tee/sed/ed/cp/mv to .env, .pem, .key, id_rsa, secrets.yml, etc.)
- [ ] Shell bypass patterns for REQUIRE_CONFIRMATION files are implemented (echo/cat/tee/sed to Dockerfile, lock files, CI configs, etc.)
- [ ] Block message goes to stderr
- [ ] Ask JSON goes to stdout
- [ ] Exit codes match specification
- [ ] Handles missing command field gracefully (exit 0)
